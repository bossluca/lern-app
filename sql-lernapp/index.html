THEORIE

Normalformen

Die drei Normalformen im Überblick

1. Normalform
Atomare Werte
Jedes Attribut enthält nur einen einzigen, unteilbaren Wert. Keine Wiederholungsgruppen, keine Listen in einem Feld.
2. Normalform
Volle Abhängigkeit
Setzt 1NF voraus. Jedes Nicht-Schlüssel-Attribut hängt vom gesamten Primärschlüssel ab – keine partiellen Abhängigkeiten.
3. Normalform
Keine transitiven Abh.
Setzt 2NF voraus. Kein Nicht-Schlüssel-Attribut darf von einem anderen Nicht-Schlüssel-Attribut abhängen.

Warum normalisieren? – Die 3 Mutationsanomalien

➕
Einfügeanomalie
Neue Daten können nicht eingefügt werden, ohne unvollständige oder redundante Einträge zu erzeugen.
✖
Löschanomalie
Beim Löschen eines Datensatzes gehen unbeabsichtigt andere wichtige Daten verloren.
✎
Änderungsanomalie
Redundant gespeicherte Daten müssen an mehreren Stellen geändert werden – Inkonsistenz ist die Folge.
Redundanz = mehrfach gespeicherte gleiche Information → führt zwangsläufig zu Dateninkonsistenz

Frage 1 aus der Klausur – Musterlösung

„Was ist das Hauptziel der Normalisierung?"

Antwort: b) Vermeidung von Redundanzen – durch Aufteilen der Daten in mehrere Tabellen werden Mehrfachspeicherungen und damit Anomalien vermieden.

GROUP BY & HAVING

Aggregatfunktionen und Gruppenauswertung – Lektion 2 & 4 von sql-tutorial.de

Auswertungsreihenfolge einer SELECT-Anweisung

FROM
→
WHERE
→
GROUP BY
→
HAVING
→
SELECT
→
ORDER BY
→
LIMIT
WHERE filtert einzelne Zeilen vor der Gruppierung – keine Aggregatfunktionen erlaubt!
HAVING filtert Gruppen nach der Gruppierung – Aggregatfunktionen sind hier Pflicht.

Aggregatfunktionen (Lektion 2)

COUNT(*)
Anzahl aller Zeilen
COUNT(Spalte)
Anzahl nicht-NULL-Werte
SUM(Spalte)
Summe der Werte
AVG(Spalte)
Durchschnittswert
MAX(Spalte)
Größter Wert
MIN(Spalte)
Kleinster Wert
Nach GROUP BY dürfen in SELECT nur Gruppierungsattribute und Aggregatfunktionen stehen. Einzelwert-Attribute erzeugen einen Fehler!

Vollständige Syntax (Lektion 4)

SELECT Region, COUNT(*) AS Anzahl, SUM(Punkte) AS Gesamt FROM Mannschaft WHERE Spiele = 10 -- filtert Zeilen (vor Gruppierung) GROUP BY Region -- gruppiert nach Region HAVING SUM(Punkte) > 30 -- filtert Gruppen (nach Gruppierung) ORDER BY Gesamt DESC LIMIT 5;

Geschachtelte SELECT-Anweisungen

Subqueries mit = und IN – Lektion 5 von sql-tutorial.de

Grundprinzip

Eine Unterabfrage steht in der WHERE-Klausel der Hauptabfrage und liefert einen Vergleichswert. So lassen sich zwei Abfragen in einer einzigen SQL-Anweisung kombinieren.

-- Muster: Unterabfrage mit Vergleichsoperator = SELECT spalte FROM tabelle WHERE spalte OPERATOR ( SELECT spalte -- NUR eine Spalte erlaubt! FROM tabelle WHERE bedingung );

Mit = Operator

Die Unterabfrage muss genau einen Wert liefern.

SELECT Name FROM Mannschaft WHERE Punkte > ( SELECT Punkte FROM Mannschaft WHERE Name = 'KBV Burhafe' );

Mit IN Operator

Die Unterabfrage darf mehrere Werte liefern.

SELECT Name FROM Mannschaft WHERE Region IN ( SELECT Region FROM Mannschaft WHERE Punkte > 15 );

Regeln auf einen Blick

1. In der SELECT-Klausel der Unterabfrage darf immer nur eine Spalte angegeben werden.
2. Wird = verwendet: Unterabfrage muss genau einen Wert zurückgeben.
3. Werden mehrere Werte erwartet: IN verwenden statt =.

Relationenmodell

Nicht-grafische Notation – Aufgabe 3 der Klausur

Notation

Format: Tabelle(PK, Attribut, FK → Tabelle)

Primärschlüssel (PK) → unterstrichen
Fremdschlüssel (FK) → gestrichelt unterstrichen + Pfeil auf referenzierte Tabelle
Bei m:n → eigene Zwischentabelle mit zusammengesetztem PK aus beiden FKs

Lösung Aufgabe 3 – Büromöbel KG

Artikel haben mehrere Bauteile; ein Bauteil kann in mehreren Artikeln vorkommen (m:n) → Zwischentabelle nötig

-- Artikel(ArtikelNr, Artikelbeschreibung, VK_Artikelpreis) -- Bauteil(BauteilID, Bezeichnung, Material) -- Mengeneinheit(MengeneinheitID, Mengeneinheit) -- Artikel_Bauteil(ArtikelNr, BauteilID, MengeneinheitID, Bauteilmenge) -- PK: (ArtikelNr, BauteilID) zusammengesetzt -- FK: ArtikelNr → Artikel -- FK: BauteilID → Bauteil -- FK: MengeneinheitID → Mengeneinheit
Hinweis aus der Aufgabe: Zwischenschritte der Normalisierung (1NF, 2NF) sind nicht notwendig – direkt zur 3NF überführen!

Aufgabe 2: Normalformen (6 P.)

a) Nennen Sie die Definitionen der drei Normalformen (NF).
b) Prüfen Sie für jede NF, ob die Tabelle (KundeID, Bestellnr, Artikel_ID, Name, Artikelname, Preis, Anmerkung) gegen sie verstößt. Begründen Sie.

Lösung anzeigen
▶
a) Definitionen:
1NF: Atomare Werte (jedes Attribut enthält nur einen einzigen, unteilbaren Wert).
2NF: 1NF erfüllt + jedes Nicht-Schlüssel-Attribut ist voll funktional abhängig vom gesamten Primärschlüssel (keine partiellen Abhängigkeiten).
3NF: 2NF erfüllt + kein Nicht-Schlüssel-Attribut hängt von einem anderen Nicht-Schlüssel-Attribut ab (keine transitiven Abhängigkeiten).

b) Überprüfung:
- 1NF: Ist verletzt! Das Feld "Anmerkung" enthält teilweise mehrere Informationen ("Auch Windows installieren", "Ubuntu aufspielen").
- 2NF: Ist verletzt! "Name" hängt nur von "KundeID" ab (Teil des Schlüssels), nicht aber von "Bestellnr". Dies ist eine partielle Abhängigkeit.
- 3NF: Ist verletzt! "Artikelname" und "Preis" hängen vom Nicht-Schlüssel-Attribut "Artikel_ID" ab (transitive Abhängigkeit).
